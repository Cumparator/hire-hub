# Hire Hub — IT Junior Aggregator

Агрегатор вакансий для начинающих: единый поиск по HH.ru и Telegram-каналам.


## 📋 Техническое задание проекта (MVP)

### 1. Общие сведения и назначение
**Hire Hub** — это агрегатор ИТ-вакансий, ориентированный на начинающих специалистов (Junior, Intern). Цель проекта — предоставить единую точку входа для поиска работы без необходимости мониторить десятки разрозненных платформ и Telegram-каналов вручную.

### 2. Функциональные требования

#### 2.1. Сбор данных (Парсинг)
Система должна в фоновом режиме (по расписанию CRON каждые 30 минут) собирать вакансии из следующих источников:
* **HeadHunter (HH.ru):** Через официальный OAuth API (основной режим) с автоматическим падением в Scraper-режим (веб-краулинг) в случае проблем с токеном. Ограничение — не более 100 вакансий на один технологический стек.
* **Другие источники (Задел на будущее):** Архитектура включает базовый класс `BaseParser`, от которого наследуются парсеры для SuperJob (`sjParser`), Зарплата.ру (`zpParser`) и др.

#### 2.2. Единая нормализация данных
Все вакансии, независимо от источника, должны приводиться к общей структуре (модели `Job`):
* Уникальный внешний идентификатор (`external_id`) и источник (`source`).
* Название, компания, описание, прямая ссылка.
* Вилка зарплат в единой валюте (по умолчанию RUB).
* Тип занятости, локация и флаг удаленной работы (`remote`).
* Категоризированный опыт работы (`intern`, `no_experience`, `between1And3`).
* Очищенный и нормализованный массив тегов технологий (стек).

#### 2.3. Поисковый движок и Фильтрация
* Полнотекстовый поиск по названию вакансий с использованием средств СУБД (индексы `GIN to_tsvector` для русского языка).
* Поиск по массиву тегов технологий (индекс `GIN` на поле `stack`).
* Гибкие фильтры: по уровню опыта, формату работы (удаленка/офис), минимальной заработной плате.

#### 2.4. Пользовательский функционал
* **Аутентификация:** Регистрация и авторизация пользователей по логину и паролю.
* **Избранное:** Возможность сохранять вакансии в закладки.


### 3. Технологический стек
* **Бэкенд:** Node.js v20 (ES Modules), Express.js, встроенный легковесный cookie-parser, модуль `pg` для работы с СУБД.
* **Фронтенд:** Vanilla JS / Сборщик Vite, модульные CSS-компоненты.
* **База данных:** PostgreSQL 16 с расширением `pgcrypto`.
* **Инфраструктура:** Docker, Docker Compose (раздельные конфигурации для локальной разработки `docker-compose.yml` и прода `docker-compose.prod.yml`).

### 4. Требования к архитектуре базы данных
СУБД должна содержать следующие таблицы:
1.  `jobs` — справочник нормализованных вакансий со строгим ограничением `UNIQUE(source, external_id)` для предотвращения дубликатов.
2.  `users` — данные пользователей и массив `favorite_job_ids.
3.  `sessions` — сессии авторизованных пользователей с контролем срока действия.
4.  `user_analytics` — сырые логи действий пользователей на платформе.
5.  `job_click_stats` — агрегированная статистика уникальных кликов пользователей по вакансиям.

## Быстрый старт

```bash
# 1. Запустить БД
docker compose up db -d

# 2. Бэкенд
cd backend && npm install && npm run dev

# 3. Фронтенд (открыть отдельный терминал)
cd frontend && npm install && npm run dev
```

Или всё сразу через Docker:

```bash
docker compose up --build
```
